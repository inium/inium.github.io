이 글은 철저히 개인 경험에 대한 글이다. 짧은 경력이지만 현재까지 소프트웨어 개발자로 살아가며 잘못한 것, 잘한 것 그리고 인상깊었던 것들에 대해 적어보고자 한다.

소프트웨어 개발자로 살아가며 잘못했던 것들과 선택을 했던 것들이 많았던 것 같다.

마치며

개발하면서 잘못했던 것들

  • framework 잘못 선택 한 것(Slim 대신 Laravel의 Lumen을 선택했어야 했다)
  • 개발 프로세스 추가 정립 하지 않은 것
  • Laravel 게시판 구현을 하면서 스캐폴딩으로 구현 하지 않은 것
  • Database view 잘못 사용한 것
  • 올해 초까지 SI 한것
  • nginx에서 php fpm 에 대한 이해를 제대로 하지 못한 것
  • 여러가지 개발했던 것들 - 성질만 늘었다
  • 서비스를 만들자는 의견을 끝까지 관철시키지 못한 것
  • 글을 너무 늦게 쓰기 시작했던 것
  • 나 자신에 대한 버퍼를 두지 않은 것

개발하면서 잘한 것들

  • Computer Graphics 선택한 것
  • Computer Graphics 개발 에서 Web 개발로 전향한 것
  • docker 도입한 것(개발/운영 단일화)
  • ubuntu 공부한 것
  • 운동을 시작한 것
  • 다른 분야의 사람들(디지털 마케팅)과 이야기 해본 것들, 그리고 그 입장에서 생각을 할 수 있게 된 것
  • webpack 도입?
  • front-end 프레임워크 하나 써본 것
  • 프로그래밍 열정?

인상깊었던 것은

  • express로 조그만 커뮤니티 구현했던 것 2번째, 3번째 준비중
  • etri 구현 (phd 장경호)
